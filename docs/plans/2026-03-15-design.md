# Creative Audio Engine - Design Document

Date: 2026-03-15

## Overview

A local web application that takes pre-generated audio files (from Gemini, Suno, etc.) and transforms them into audiophile-quality, intelligently arranged, infinitely looping tracks. Features an RL-powered creative engine for arrangement decisions, a multi-model enhancement pipeline, and a dedicated sleep mode with binaural/isochronal entrainment.

Not a music generator. A creative engine for existing audio.

## Hardware Constraints

- CPU: Intel i7-1255U (12 threads, AVX2/VNNI)
- RAM: 24GB
- GPU: Intel Iris Xe integrated only (no CUDA, no ROCm)
- All ML inference runs on CPU via ONNX Runtime
- Storage: 93GB free on NVMe SSD

## Enhancement Pipeline

Each stage has a model selector dropdown with estimated processing time.

1. Analysis (librosa): BPM, key, energy, beat grid, segments, self-similarity
2. Stem Separation (Demucs): htdemucs | htdemucs_ft | htdemucs_6s
3. Per-Stem Enhancement:
   - Noise Reduction: DeepFilterNet | noisereduce
   - Super-Resolution: FlashSR (fast) | AudioSR (quality) | AERO (spectral)
4. Remix & Spatial: stem recombination, HRTF binaural, stereo width
5. Mastering: Matchering (reference) | dasp-pytorch (learned), pyloudnorm, Pedalboard limiting
6. Export: FLAC/WAV/ALAC, 44.1/48/96kHz, 16/24/32-bit

Every stage emits progress via WebSocket. Every intermediate result is playable. A/B between any stage and original.

## RL Creative Engine

DPO-based arrangement agent.

State: audio features (spectral centroid, RMS energy, tempo, key, harmonic content, mood), position in mix, previous transition, target mood curve.

Actions: select next track, set crossfade duration (1-30s), choose transition type (linear, equal-power, beat-synced, stem-staggered), per-stem transition offsets, EQ at transition, loop region boundaries.

Reward (multi-signal): FAD score, SongEval aesthetic rating, beat alignment accuracy, key compatibility, energy curve smoothness, user thumbs up/down feedback.

Training: DPO - generate two candidate arrangements, evaluate both, learn to prefer the better one. User feedback personalizes over time.

UI: "Generate Mix" button, agent proposes arrangement on timeline, accept/tweak/regenerate, confidence scores per transition, lock segments and rearrange rest.

## Seamless Looping

1. Loop Point Detection: self-similarity matrix from chroma features, constrain to beat boundaries, score by harmonic + energy + spectral match
2. Loop Transition: equal-power crossfade, adaptive length (2-8 bars), per-stem crossfade if available, zero-crossing alignment
3. Validation: render loop transition, FAD check, fallback to next candidate
4. Export: single file with loop markers, continuous rendered file (1-8hr), browser gapless playback

## Sleep Mode

Dedicated full-screen mode, separate from studio view.

### Audio Output Detection
- Headphones: binaural beats + HRTF spatial
- Speakers: isochronal tones + bass boost + room fill

### Frequency Entrainment
- Binaural beats: two sine waves with Hz offset per ear (headphones only)
- Isochronal tones: amplitude-modulated pulse (works on speakers)
- Pink/brown noise: generated, infinite, never repeats

### Sleep Stage Presets
- Wind Down: 10Hz alpha -> 6Hz theta (30 min ramp)
- Deep Sleep: 2Hz delta steady
- Full Cycle: alpha -> theta -> delta -> theta -> alpha (90 min)
- Custom: user sets start/end frequency + duration

### Sleep Timer
- Presets: 30m, 1hr, 2hr, 4hr, 8hr, endless
- Gradual volume fadeout over configurable duration
- Alarm: gradual volume ramp-up with delta -> alpha frequency shift

### Speaker Features (Z623 / subwoofer systems)
- Bass boost profile for subwoofer
- Room fill mode (stereo widening)
- Brown noise / ambient texture underlayer

### Headphone Features
- Binaural beats
- HRTF spatial positioning
- Intimate mixing (reduced stereo width)

### Sleep Transition Effect
On activating sleep mode:
1. UI elements fade out (0.5s)
2. Cinematic effect plays (3-5s), user-selectable:
   - Ripple: waveform pulses outward like water, dissolves to black
   - Particles: spectrogram breaks apart, drifts upward like embers
   - Horizon: waveform flattens to a line, pulses, fades
3. Music crossfades in (5s), entrainment layer fades in (10s)
4. Pure black screen (#000000), display dims to minimum
5. Tap anywhere: clock + transport fade in for 5s, auto-hide

### Sleep UI
- Almost nothing visible, dark as possible
- Optional: slow breathing animation synced to entrainment frequency
- Optional: fully black (no light at all)
- Tap-to-reveal clock and transport

## Implementation Phases

### Phase 1: Foundation
- FastAPI server, file upload, audio analysis (librosa)
- Redis + Celery setup
- React app shell, Zustand stores, layout
- Library panel: upload, display BPM/key/energy
- WaveSurfer waveform rendering
- Audio playback via Web Audio API
- WebSocket connection
- Tests: pytest, vitest, Playwright smoke test

### Phase 2: Enhancement Pipeline
- Demucs stem separation (all variants)
- FlashSR + AudioSR super-resolution (model selector)
- DeepFilterNet denoising
- Mastering chain (Matchering + pyloudnorm + Pedalboard)
- Pipeline panel UI with model dropdowns, progress rings
- A/B toggle: original vs processed
- WebGL spectrogram with before/after split slider
- Tests: audio validation, E2E pipeline flow

### Phase 3: Timeline & Arrangement
- Multi-track timeline with drag-from-library
- Beat grid overlay
- Crossfade regions with adjustable curves
- Loop point detection and markers
- Mix rendering to single output
- Export to FLAC/WAV at configurable quality
- Tests: E2E drag, arrange, export, verify output

### Phase 4: RL Creative Engine
- State representation from analysis
- Action space (track selection, crossfade, transitions)
- Reward functions (FAD, beat alignment, key, energy)
- DPO training loop
- "Generate Mix" button, confidence scores
- Lock/unlock, user feedback
- Tests: RL agent produces valid arrangements, transitions sound correct

### Phase 5: Sleep Mode
- Binaural/isochronal/noise generators
- Headphone vs speaker detection
- Sleep presets, timer, fadeout, alarm
- Bass boost, room fill for speakers
- Full-screen sleep UI
- Three transition effects (ripple, particles, horizon)
- Tap-to-reveal controls
- Tests: E2E sleep mode activation, audio output verification, timer accuracy

### Phase 6: Polish & Ship
- HRTF spatial positioning
- SVG animations throughout
- Keyboard shortcuts
- Drag-and-drop file import anywhere
- Error handling and edge cases
- Docker one-command setup
- README with screenshots and spectrograms

## Testing Strategy

Every phase includes:
- Unit tests (pytest backend, vitest frontend)
- Integration tests (API endpoints with real audio files)
- E2E tests (Playwright - clicking buttons, dragging, verifying audio)
- Audio validation (correct format, sample rate, duration, not silent, not clipped)
- Manual smoke testing through browser

No feature ships without verified automated tests passing.
